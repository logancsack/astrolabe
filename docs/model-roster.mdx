Astrolabe uses a static checked-in roster. Runtime routing does not depend on live discovery.

OpenRouter is the maintainer reference for model IDs and pricing, but the active roster is whatever Astrolabe ships in `src/manifests/index.js`.

## Roster policy

- Each model must earn an active slot on cost, capability, and recency. Models more than two generations old should be raw-only unless they provide a uniquely cheap role.
- `deepseekV4Pro` is the first serious-work default.
- `m27` remains the conservative fallback and safety-floor workhorse.
- `gpt-5.4-nano` owns the active nano/control alias.
- Kimi, GLM, Qwen, MiMo, and Gemma stay first-class specialists where they win on benchmarks or price.
- Preview and experimental models are opt-in and excluded from default lane manifests.
- Rejected older models stay in `raw_only` for one transition release, then should be removed unless a maintainer re-approves a manual-pin reason.
- Raw model additions should be checked into the manifest explicitly.
- Use `npm run validate:models` to compare the static manifest against OpenRouter before releasing roster changes.
